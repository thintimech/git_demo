# 简化驾培系统开题报告（项目总览）

## 1. 已完成工作、课题研究主要内容、目的与意义

### 1.1 课题研究主要内容
本课题围绕“学员—教练—管理员”三类角色，设计并实现一个前后端分离的简化驾培系统，覆盖以下核心业务链路：
1. 学员注册登录与身份认证；
2. 题库练习、视频学习、公告与消息通知；
3. 教练排班发布与学员预约训练；
4. 训练记录、考试记录、聊天记录等过程数据沉淀；
5. 管理端对学员、教练、套餐与排班的统一管理。

### 1.2 已完成的主要工作（截至当前）
1. **完成系统总体架构搭建**：后端采用 Spring Boot + Spring Security + JPA，前端采用 React + TypeScript + Vite，实现前后端分离与统一 API 交互。
2. **完成三角色权限体系**：基于 JWT 与 RBAC 实现 STUDENT/COACH/ADMIN 访问控制，形成接口级与页面级双重权限约束。
3. **完成核心业务模块开发**：
   - 学员/教练/管理员管理；
   - 题库练习与学习统计；
   - 训练记录、考试记录、聊天记录；
   - 教学视频、公告中心、通知中心。
4. **完成训练预约主链路升级**：
   - 管理员标注学员套餐是否已购；
   - 学员绑定唯一教练；
   - 学员仅查看已绑定教练未来一周排班并预约；
   - 保持“教练确认后扣减课时”的业务规则。
5. **完成前端角色页面职责分离与体验优化**：将多角色混用页面拆分为独立页面，并统一引入可视化空状态组件（EmptyStateCard），提升可用性与一致性。
6. **完成阶段性验证**：后端测试与前端构建均可通过，具备继续论文实验与答辩材料整理的基础。

### 1.3 课题研究目的
1. 构建一套可运行、可验证、可扩展的驾培管理原型系统；
2. 提升传统驾培流程中“信息分散、预约不透明、协同效率低”的问题处理能力；
3. 通过标准化接口、角色化流程与数据留痕，为后续智能分析与管理优化提供数据基础。

### 1.4 课题研究意义
1. **实践意义**：推动驾培业务流程数字化，提升管理效率与服务体验。
2. **工程意义**：验证前后端分离、权限控制、事务一致性、接口规范化在真实业务中的落地路径。
3. **教学与学术意义**：形成完整的软件工程案例（需求分析—设计—实现—测试—总结），为毕业设计提供可展示、可复现实验成果。

## 2. 拟采取的研究方法（方案、技术路线）与可行性研究（可提交系统描述）

### 2.1 研究方法与实施方案
本课题拟采用“**需求分析 + 迭代实现 + 测试验证 + 文档固化**”的方法开展。首先通过业务流程梳理明确学员、教练、管理员三方职责边界；其次进行领域建模与数据库结构设计，建立统一的数据实体与接口契约；随后采用前后端并行迭代开发方式逐步实现各功能模块，并通过单元测试、接口测试和场景联调验证业务正确性；最后对系统进行可用性优化与论文材料整理，完成成果固化。

### 2.2 技术路线
1. **后端层面**：
   - Spring Boot 构建 REST 服务；
   - Spring Security + JWT 实现认证鉴权；
   - Spring Data JPA 完成数据访问与事务管理；
   - MySQL/H2 支撑开发与测试场景。
2. **前端层面**：
   - React + TypeScript 构建页面；
   - React Router 实现角色路由隔离；
   - React Query 管理异步请求与状态同步；
   - 统一组件化样式提升页面一致性与可维护性。
3. **质量保障层面**：
   - Maven 测试与前端构建作为阶段验收手段；
   - 关键业务链路采用“接口校验 + 角色校验 + 状态校验”三重约束。

### 2.3 可行性研究
1. **技术可行性**：所用框架成熟、生态完善，团队（个人）具备 Java Web 与 React 开发基础，现有原型已跑通关键链路。
2. **业务可行性**：系统流程贴合驾培实际场景（报名、排班、预约、记录、通知），需求边界清晰，便于持续迭代。
3. **数据可行性**：数据库模型已建立，实体关系完整，能够支撑预约、记录、通知等核心业务数据。
4. **进度可行性**：采用九周分阶段推进，任务可拆解、可验收、风险可控。
5. **风险与对策**：
   - 角色权限复杂导致越权风险：通过接口与页面双重鉴权规避；
   - 预约并发导致状态不一致：通过事务与锁机制保障一致性；
   - 后期联调延期：采用每周里程碑与持续构建提前暴露问题。

## 3. 预期结果

1. 形成一套可部署运行的“简化驾培系统”原型，覆盖学员、教练、管理员三端核心功能。
2. 实现完整训练预约闭环：**套餐状态管理 → 绑定教练 → 周排班预约 → 教练确认扣减**。
3. 建立规范化接口与权限体系，保证关键业务在多角色下的正确性与安全性。
4. 输出系统设计与实现文档，包括：需求说明、技术路线、数据库设计、关键流程说明、测试结果与总结。
5. 完成毕业论文与答辩材料，形成可展示的工程化成果（系统演示 + 文档证明 + 测试依据）。
